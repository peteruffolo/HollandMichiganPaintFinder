# Pigment — cheapest oil paint finder

Finds the cheapest place to buy a given oil color across your stores, built for
**in-person shopping** in Holland, MI (with Blick as a Chicago-trip / online option).
Matches color names loosely and uses Colour Index pigment codes to tell a genuine
color from its "permanent" replacement.

## How it works

Two halves, because a browser page (GitHub Pages) can't scrape other sites itself:

```
catalog.json          canonical products: brand, line, color, PIGMENTS, sizes
                       (rarely changes; pigments come from the manufacturer)
        +
prices/*.json         one file per distributor: price, size, stock, url
                       (changes often; produced by scrapers or hand-entered)
        |
   build_data.py       joins them ->
        v
data.json             flat list the web page reads
        |
   index.html          the search UI (deploy to GitHub Pages)
```

The scrapers run on **your machine or a GitHub Action** (not in the browser), so
there's no cross-origin problem. Pigment data lives only in `catalog.json`, so a
distributor that doesn't list pigments (most of them) is fine.

## Repo map

| Path | What it is |
|------|------------|
| `index.html` | The search page. Open locally, or host on GitHub Pages. |
| `catalog.json` | Master product list + pigment codes. **Expand this over time.** |
| `prices/fris.json` | Auto-generated by the Fris scraper. |
| `prices/manual.json` | Hand-entered prices (Hope Bookstore, etc.). Edit by hand. |

| `scrapers/common.py` | Shared name-to-catalog matching. |
| `scrapers/fris_scraper.py` | Fris scraper. **Ready.** |
| `scrapers/michaels_scraper.py` | Michael's (Playwright, store #4729). **Scaffold.** |
| `.github/workflows/update-prices.yml` | Scheduled auto-update. |

## Quick start

```bash
pip install requests beautifulsoup4
python scrapers/fris_scraper.py     # writes prices/fris.json
python build_data.py                # writes data.json
# open index.html in a browser
```

To publish: push to GitHub, then Settings -> Pages -> deploy from the `main` branch.
The page is then at `https://<you>.github.io/<repo>/`. No custom domain needed.

## Distributor status

- **Fris Supply Shop** — ready. Server-rendered prices; carries Gamblin 1980 & W&N Winton.
- **Hope College Bookstore** — manual. No online prices. Edit `prices/manual.json`;
  each entry has a `verified` date and the UI flags it stale after 45 days.
- **Michael's (Holland #4729)** — scaffolded. Online price = shelf price and
  "pickup today" = in stock, so it's automatable. Needs one tuning run to confirm
  the price/pickup CSS selectors (see notes at the top of `michaels_scraper.py`),
  plus a `michaels_products.json` listing the URLs of colors you buy.
- **Blick** — not yet. Product pages expose pigment data in HTML (great for
  *building the catalog*), but price is JS-rendered, so it needs Playwright like
  Michael's. Lower priority since it's your trip/online option.

## Expanding the catalog

`catalog.json` is the foundation. The fastest way to fill it: Blick product pages
list each color's pigment codes, color name, and sizes in plain HTML. Entries
marked `_note: "VERIFY ..."` are best-guesses to replace with manufacturer data.

Each entry:
```json
{
  "id": "gamblin-1980-alizarin-permanent",
  "brand": "Gamblin", "line": "1980", "grade": "student",
  "name": "Alizarin Permanent",
  "aliases": ["permanent alizarin crimson", "alizarin crimson"],
  "pigments": ["PR177"], "hex": "#90203b", "sizes": [37, 150]
}
```

`grade` drives the Student/Artist filter. `aliases` improve matching across the
naming variations distributors use. `pigments` power the genuine-vs-permanent logic.
